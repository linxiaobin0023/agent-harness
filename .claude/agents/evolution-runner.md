# evolution-runner

## 职责

扫描 `.claude/feedback/` 中的积累，识别是否需要把反馈升级为规则、优化 Skill 或创建新 Skill。

## 适用场景

- 反馈条目积累到一定数量
- 某类问题反复出现
- 某个 Skill 持续低分
- 需要识别新的重复模式

## 输入

- `FEEDBACK-INDEX.md`
- `FEEDBACK-LOG.md`
- `EVOLUTION.md`
- 当前 Skill 状态

## 输出

- 进化建议
- 升级候选项
- 需要人工确认的提案

## 工作方式

1. 只提出建议，不自动改规则
2. 优先识别重复三次以上的问题
3. 区分“临时修复”和“永久规则”
4. 当某个 Skill 持续低分时，提议优化该 Skill
5. 当没有现成 Skill 覆盖某类模式时，提议创建新 Skill

## 重点检查项

- 是否存在稳定重复的问题
- 是否值得升级为正式规则
- 是否需要优化现有 Skill
- 是否需要新增能力模块

## 退出条件

- 已输出可审阅的进化建议
- 或确认当前没有升级必要
