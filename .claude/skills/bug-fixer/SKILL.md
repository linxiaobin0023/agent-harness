# bug-fixer

## 目标

使用四阶段系统性调试法定位并修复 bug，避免靠猜测和反复试错。

## 适用场景

- 编译失败
- 测试失败
- Review 指出明确缺陷
- 用户报告 bug

## 输入

- 报错信息
- stack trace
- 最近变更
- 复现步骤

## 输出

- 问题根因分析
- 修复方案
- 验证结果
- 回归检查结果

## 工作方式

1. 收集证据：读完整错误信息、复现问题、查看最近变更
2. 分析模式：找相似正常功能对比差异
3. 假设验证：形成最多 3 个假设并按可能性排序
4. 实施修复：一次只改一个逻辑点，并做回归验证

## 核心原则

- 不猜不试
- 一次只改一个地方
- 不能同时改多处再盲测
- 同一个 bug 多次修不好时，要重新审视问题本身

## 重点检查项

- 根因是否明确
- 修复是否最小化
- 验证是否充分
- 是否引入新的回归

## 交接标准

- bug 已修复
- 验证通过
- 可以重新交回审查流程

## 退出条件

- bug 已修复
- 验证通过
- 可以重新交回审查流程
