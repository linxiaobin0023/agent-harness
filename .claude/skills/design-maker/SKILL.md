# design-maker

## 目标

根据 `Product-Spec.md` 和 `Design-Brief.md` 生成可执行的设计图或原型图方案，并确保 UI 表达与需求、规范一致。

## 适用场景

- 设计规范已确认，需要生成页面原型
- 需要把信息结构和交互流程可视化
- 需要在实现前锁定视觉与布局

## 输入

- `Product-Spec.md`
- `Design-Brief.md`
- 页面结构
- 交互流程

## 输出

- 设计图说明
- 原型图生成提示
- 页面层级与布局建议

## 工作方式

1. 优先对齐设计图，其次是 Design-Brief，最后才是 Spec
2. 使用 Pencil 或 Figma 的 MCP 能力生成页面原型
3. 对关键页面、关键状态、关键交互进行覆盖
4. UI 变更必须同步回写设计文档

## 问题维度

- 页面结构是否完整
- 关键交互是否被覆盖
- 设计是否与规范一致
- 是否存在未建模的 UI 场景

## 重点检查项

- 设计是否能指导开发
- 页面是否有明确层级
- 状态是否覆盖完整
- 是否存在歧义布局

## 交接标准

- 设计图已足够指导开发
- 页面结构已稳定
- 已准备好交给 `dev-planner`

## 退出条件

- 设计图已足够指导开发
- 页面结构已稳定
- 可以进入开发计划阶段
