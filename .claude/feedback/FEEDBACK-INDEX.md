# FEEDBACK-INDEX

## 目的

快速索引历史反馈，方便新 session 读取已有问题、状态和处理结果。

## 索引规则

- 每条反馈都要有唯一编号
- 记录来源、问题类型、状态、关联文件
- 只保留可追踪、可复盘的信息
- 索引必须能快速定位到原始记录

## 索引格式

- F-YYYYMMDD-001
  - 类型：
  - 来源：
  - 状态：
  - 关联文件：
  - 简述：

## 状态建议

- 新建
- 处理中
- 已解决
- 已升级
- 已关闭

## 快速加载要求

- 新 session 启动时应优先读取索引
- 需要能快速判断哪些反馈仍待处理
- 需要能快速找到最近的高频问题
