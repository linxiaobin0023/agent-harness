# EVOLUTION

## 目的

定义反馈如何升级成规则、Skill 优化或新 Skill 创建提案，并让这些提案能形成稳定的 Steering Loop。

## 进化层级

### 1. 记录

静默收集反馈，写入 `FEEDBACK-LOG.md`。

### 2. 毕业

同类反馈出现三次以上，提议升级为正式规则。

### 3. 优化

某个 Skill 持续低分，提议优化该 Skill。

### 4. 创建新 Skill

某种重复模式没有现成 Skill 覆盖时，提议创建新的 Skill。

## 规则

- 不自动改规则
- 所有升级提议都需要人工确认
- 优先解决高频、重复、可验证的问题
- 进化提案必须能追溯到具体反馈

## 处理流程

1. 收集反馈
2. 归类问题
3. 统计重复频次
4. 形成提案
5. 等待人工确认
6. 确认后再执行

## 输出要求

- 提案要可审阅
- 规则要可追踪
- 优化要可验证
- 新 Skill 要有明确触发条件
