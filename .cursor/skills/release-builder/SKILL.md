# release-builder

## 目标

完成构建、打包、发布前检查和交付说明，确保发布结果可追踪、可验证、可回滚。

## 适用场景

- 代码通过审查
- 需要打包交付
- 需要生成发布说明和回滚方案

## 输入

- 已通过审查的代码
- 发布目标
- 环境配置
- 变更记录

## 输出

- 构建结果
- 发布说明
- 验证步骤
- 回滚建议

## 工作方式

1. 先确认构建通过
2. 再输出发布材料
3. 发布信息必须可追踪、可回滚
4. 若发布失败，明确失败点并返回修复流程

## 重点检查项

- 构建是否成功
- 发布配置是否正确
- 是否有回滚预案
- 是否完成交付说明

## 交接标准

- 构建与发布准备完成
- 发布材料已生成
- 可以进入交付或上线阶段

## 退出条件

- 构建与发布准备完成
- 发布材料已生成
- 可以进入交付或上线阶段
