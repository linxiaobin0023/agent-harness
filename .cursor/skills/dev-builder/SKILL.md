# dev-builder

## 目标

按 `DEV-PLAN.md` 把项目拆成一个个 Task 逐步实现，并在每个 Task 后做局部验证。

## 适用场景

- `DEV-PLAN.md` 已完成
- 需要开始写代码
- 需要按 Task 有序推进实现

## 输入

- `DEV-PLAN.md`
- 当前 Task 描述
- 相关源码文件
- 相关约束与依赖

## 输出

- 代码实现
- 局部编译结果
- Task 完成记录
- 待审查状态标记

## 工作方式

1. 每次只处理一个 Task
2. 先读清楚上下文，再动手实现
3. 不得跨 Task 预先实现后续内容
4. 每完成一个 Task，必须先做编译或运行验证
5. 如果出现错误，优先修复当前 Task 引入的问题
6. 不允许用模糊结论代替验证结果

## 问题维度

- 当前 Task 是否完成
- 当前 Task 是否通过验证
- 是否引入了新的副作用
- 是否可以安全交给审查

## 重点检查项

- 代码是否符合任务要求
- 是否存在未完成的边界情况
- 是否有可见的运行错误
- 是否需要进入 `code-review`

## 交接标准

- 当前 Task 实现完成
- 当前 Task 验证通过
- 已准备好交给 `code-review`

## 退出条件

- 当前 Task 实现完成
- 当前 Task 验证通过
- 已准备交给 `code-review`
