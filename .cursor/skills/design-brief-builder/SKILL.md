# design-brief-builder

## 目标

把抽象的视觉偏好量化为可执行的 `Design-Brief.md`，确保后续设计与实现拥有统一的视觉标准。

## 适用场景

- Product-Spec 已确认，但视觉方向还不明确
- 用户提出风格偏好、品牌约束、界面感觉
- 需要把抽象审美变成可落地规则

## 输入

- `Product-Spec.md`
- 风格偏好
- 品牌约束
- 参考设计

## 输出

- `Design-Brief.md`
- `Design-Brief-CHANGELOG.md`
- 设计约束清单

## 工作方式

1. 先确认总体风格，再拆配色与排版
2. 将模糊形容词转成可执行规则
3. 让后续设计和开发都能统一对齐
4. 需求变化时先更新 Design Brief，再进入实现

## 问题维度

- 风格关键词是否明确
- 配色方案是否清晰
- 组件风格是否统一
- 状态和交互是否可落地
- 是否符合产品定位

## 重点检查项

- 视觉约束是否可执行
- 设计语言是否一致
- UI 状态是否完整
- 是否与需求目标冲突

## 交接标准

- 视觉方向已明确
- `Design-Brief.md` 已能指导设计图制作
- 已准备好交给 `design-maker`

## 退出条件

- 视觉方向已明确
- 设计约束已稳定
- 可以进入设计图制作阶段
