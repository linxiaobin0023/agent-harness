# Cursor 流程映射

## 核心流程

需求 → 设计规范 → 设计图 → 开发计划 → 开发 → 审查 → 修复 → 发布 → 反馈 → 进化

## 文档映射

- 需求阶段 → `Product-Spec.md`
- 设计阶段 → `Design-Brief.md`
- 计划阶段 → `DEV-PLAN.md`
- 开发阶段 → 当前代码文件
- 审查阶段 → 当前代码文件 + 相关文档
- 修复阶段 → 当前错误文件 + 最近变更
- 发布阶段 → git 状态 + 提交内容
- 反馈阶段 → `.claude/feedback/` 或 `.cursor/feedback/`
- 进化阶段 → `EVOLUTION.md`

## 工具行为

- 读项目文档时，只读当前阶段相关文件
- 改动前先确认阶段
- 每步完成后立即验证
- 出问题时回到对应阶段修复
- 提交前检查仓库状态
- 推送失败先排查网络或远程配置
